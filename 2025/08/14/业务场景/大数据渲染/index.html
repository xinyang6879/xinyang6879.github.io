懒加载

使用IntersectionObserver判断元素是否显示

  • 当元素初次在页面上显示, 以及之后每次显示和隐藏都会触发回调

  • IntersectionObserver API 是异步的，不随着目标元素的滚动同步触发。

  • 只有线程空闲下来，才会执行观察器。这意味着，这个观察器的优先级非常低，只在其他任务执行完，浏览器有了空闲才会执行。

虚拟滚动

原理是：用一个大的div撑出所需要的高度实现滚动条，然后元素通过transform: translateY()显示元素。

  1. 切片渲染（Windowing）

    • 我们有一个容器（container），它有一个固定的高度（clientHeight），这个高度决定了可视区域的大小。
    • 容器内部有一个“垫片”或“占位符”元素（placeholder），它的高度等于所有列表项的总高度（itemSize * itemCount）。这个元素的作用是撑开容器，让滚动条的行为和拥有全部数据的列表一样。
    • 容器内部还有一个列表（list），它负责实际渲染当前需要显示的项。它的位置是通过 transform: translateY(...) 来动态调整的。

  2. 计算可视区域（Calculation）

    • 监听容器的 scroll 事件（或使用 Intersection Observer API）。
    • 当用户滚动时，根据容器的 scrollTop（滚动距离）来计算：
      • 起始索引（startIndex）：当前应该从第几条数据开始渲染。
        • startIndex = Math.floor(scrollTop / itemHeight)
      • 结束索引（endIndex）：当前应该渲染到第几条数据结束。通常会多渲染一些（例如前后多渲染 5 条），以防止滚动时出现空白。
        • endIndex = startIndex + Math.ceil(clientHeight / itemHeight) + bufferSize
    • 根据 startIndex 和 endIndex，从完整的数据列表中“切片”出需要显示的数据（visibleData = allData.slice(startIndex, endIndex)）。

  3. 动态渲染与定位（Rendering & Positioning）

    • 将“切片”得到的 visibleData 渲染到页面上。
    • 同时，计算列表的偏移量（offset）。因为起始索引之前还有很多未渲染的项，我们需要将列表向下平移，让当前渲染的项出现在正确的位置。
      • offset = startIndex * itemHeight
    • 将列表元素设置为 style="transform: translateY(${offset}px)"。

  4. 回收与复用（Recycling）

    • 当用户滚动时，会不断地计算新的 startIndex 和 endIndex。
    • 之前渲染的 DOM 节点不会被销毁，而是被复用来显示新的数据内容。只是更新它们内部的内容（如文本、图片），这比创建和销毁 DOM 节点的性能开销要小得多。

四、关键优化与注意事项

  1. 定高 vs 不定高：

    • 定高：实现最简单，计算性能最好（如上例）。几乎所有虚拟滚动库都首选支持定高。
    • 不定高：实现更复杂。需要预先知道或动态测量每一项的高度，通常需要维护一个“位置索引表”（positionCache）来记录每个索引的偏移量，计算量更大。

  2. 缓冲区（Buffer）：

    • 在可视区域之外多渲染一些项目，可以防止在快速滚动时出现短暂的空白。

  3. 使用绝对定位或 Transform：

    • 使用 transform: translateY() 来移动列表通常比使用 top 进行绝对定位性能更好，因为它不会触发重排（Reflow），只触发重绘（Repaint）。

  4. 现代 API：

    • 优先使用 Intersection Observer 来替代 scroll 事件监听，因为它更高效，并且是在主线程之外异步触发的。

  5. 框架集成：

    • 在 React 中，有 react-window 和 react-virtualized 等著名库。
    • 在 Vue 中，有 vue-virtual-scroller 等库。
    • 它们都封装了上述核心逻辑，提供了简单易用的组件。
